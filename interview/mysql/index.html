什么是MySQL

一种开源的关系型数据库, 使用频率很高

怎么删除/创建一张表和设定主键

删除

drop 表名 from 数据库;

创建

首先指定数据库

use database;

create 表 {
	列名 数据类型;
	列名 数据类型;
	列名 数据类型;
}

设定主键

primary key (列名);

sql升序排序

select 字段 from 数据库 order by desc/asc

此外，如果我需要根据多个字段进行排序，例如先按工资降序排列，再按名字升序排列，我可以使用：

SELECT id, name, salary
FROM employees
ORDER BY salary DESC, name ASC;

MySQL 的内连接、左连接、右连接有什么区别?

内联, 返回两个表的交集 左联, 返回左表所有记录, 右表匹配返回, 没有匹配的值填空 右联是左联把表换个顺序

数据库三大范式

第一范式, 表的每一列都是不可分割的基本单元

地址作为省市区的集合应该拆成四个字段

第二范式, 在第一范式的基础上, 消除部分依赖

所有非主键字段都应该依赖主键, 而不能只与主键的某一部分相关。

比如订单ID和商品ID和商品名, 单价

应该将其拆分为订单表、订单商品关联表、商品表。

第三范式. 在第二范式的基础上消除传递依赖

非主键列应该只依赖于主键列，不依赖于其他非主键列。

设计订单信息表的时候，可以把客户名称、所属公司、联系方式等信息拆分到客户信息表中，然后在订单信息表中用客户编号进行关联。

建表时注意哪些问题?

在建表的时候，首先可以考虑表是否符合数据库范式，也就是确保字段不可再分，消除非主键依赖，确保字段仅依赖于主键。

然后在选择字段类型时，尽量选择合适的数据类型。

在字符集上，尽量选择 utf8mb4，这样不仅可以支持中文和英文，还可以支持表情符号等。

当数据量较大时（比如上千万行数据），需要考虑分表。比如订单表，可以采用水平分表的方式来分散存储压力。

Varchar 和 char的区别

字符串, 固定空间, 小于固定空间的用空格填充 速度快, 空间换时间

可变字符, - 插入的数据是多长，就按照多长来存储； 时间换空间, 存取慢, 但占用空间小

DATETIME 和 TIMESTAMP 的异同？

都可以存储时间 YYYY-MM-DD HH:MM:SS

日期范围：DATETIME 的日期范围是 1000-01-01 00:00:00.000000 到 9999-12-31 23:59:59.999999 无时区

TIMESTAMP 的时间范围是1970-01-01 00:00:01.000000 UTC 到 ``2038-01-09 03:14:07.999999 有时区

存储空间：DATETIME 的存储空间为 8 字节；TIMESTAMP 的存储空间为 4 字节

in 和 exists 的区别？

  1. 如果查询的两个表大小相当，那么用 in 和 exists 差别不大。

drop、delete 与 truncate 的区别？

三者都是删除 drop从库中删除表, 所有记录,索引权限

count(1)、count(*) 与 count(列名) 的区别？

记录货币用什么好

Decimal

DECIMAL 和 NUMERIC 值作为字符串存储，而不是作为二进制浮点数，以便保存那些值的小数精度

记录Emoji有什么好?

emoji是4个字节的UTF-8字符 而utf-8字符集最多支持3个字节
